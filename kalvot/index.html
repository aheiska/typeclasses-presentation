Real signature of List.sorted is: 


def sorted[B >: A](implicit ord: math.Ordering[B]): List[A]

What is that implicit parameter. OMG Scala is so complicated!

What is math.Ordering

A type class!

So what is Ordering[Int] or Ordering[Foo] then

Obligatory theory slide

Typeclass can be though as a way to add more functionality to an existing class. But they can express more than that. Just like normal classes they can have an inheritance hierarcy and writing a sub instance can gives you combatibility with all algorithms requiring super-instance.

Good for anything else than sorting?

For plenty of things! Even for things that would be difficult to express with classes or interfaces.

More generic algorhitms

Using typeclasses one does not need to have access to the original types to be able to make them implement the typeclass interface.

Multiple instances for same type

(This would be impossible with interfaces / traits)

There can also be multiple implementations (in scala), but that is not something that should actually be used. (This can be confusing and error prone, use with care).

More expressive power

Typeclasses enable more complex abstractions which would be difficult and clunky to accomplish using classes.

With typeclasses it is possible to pair operation and a type with an abstract way. (E.g. Monoid)
